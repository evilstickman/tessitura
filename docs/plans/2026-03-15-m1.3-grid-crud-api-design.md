# M1.3: Grid CRUD API — Design

## Goal
Build the first API endpoints for Tessitura: CRUD operations for practice grids. This milestone establishes the MCV patterns that all future endpoints follow.

## Maps to
- UC-1.3 (Create Practice Grid)
- UC-1.12 (View Practice Grid)
- UC-1.13 (Delete Practice Grid)

## Scope
- Four REST endpoints (create, list, detail, delete)
- MCV layer structure (model, controller, view, route)
- Auth placeholder for pre-M1.8 development
- Schema migration: add `passageLabel` to PracticeRow
- 42 test cases (unit + integration)

## Schema Changes

### Add `passageLabel` to PracticeRow

```prisma
passageLabel String? @map("passage_label")
```

Optional String field on PracticeRow. When present, the UI displays it instead of "mm. X-Y" (e.g., "Letter C to D", "Intro"). The existing `startMeasure` (Int) and `endMeasure` (Int) remain as canonical data — `passageLabel` is a human-friendly override for display purposes.

Requires a Prisma migration. Also requires updating UC-1.4 in PRODUCT_SPEC.md to document this field.

## Auth Placeholder

Authentication is M1.8. Until then, we use a contained placeholder:

- **`src/lib/auth.ts`** — exports `getCurrentUserId(): string`, returns the seed user's ID
- **`prisma/seed.ts`** — creates a dev user with email `dev-placeholder@tessitura.local`
- `package.json` gets `prisma.seed` config so `npx prisma db seed` works
- Seed script is dev/test only — never runs in CI or production
- Every controller calls `getCurrentUserId()` — no hardcoded user IDs leak into business logic

**Containment guarantees:**
- Single function to replace at M1.8 — no scattered references
- Seed user has an obviously-fake email that screams "not real"
- M1.8 acceptance criteria must include: "no references to seed user ID remain outside of seed script"

## API Endpoints

Four endpoints under `/api/grids`. REST conventions, JSON request/response.

| Method | Path | Purpose | Status |
|--------|------|---------|--------|
| POST | `/api/grids` | Create grid | 201 / 400 |
| GET | `/api/grids` | List user's grids | 200 |
| GET | `/api/grids/[id]` | Grid detail (full nested) | 200 / 400 / 404 |
| DELETE | `/api/grids/[id]` | Soft delete + cascade | 204 / 400 / 404 |

No PUT/PATCH — grid editing is not specified in M1.3.

### POST /api/grids — Create Grid
- **Body:** `{ name: string, notes?: string }`
- **Validation:** name required, 1-200 chars, trimmed, not whitespace-only
- **Behavior:** sets `userId` from `getCurrentUserId()`, `fadeEnabled` defaults to `true`
- **Response:** 201 with created grid (formatted via view)

### GET /api/grids — List Grids
- **Scoping:** current user only, excludes soft-deleted (handled by Prisma extension)
- **Sorting:** `updatedAt` descending (most recently modified first)
- **Response:** 200 with array of formatted grids (summary — no nested rows)

### GET /api/grids/[id] — Grid Detail
- **Ownership:** 404 if grid doesn't belong to current user (not 403, avoids leaking existence)
- **Nesting:** full depth — grid → rows → cells → completions
- **Sorting:** rows by `sortOrder` asc, cells by `stepNumber` asc, completions by `completionDate` asc
- **Response:** 200 with nested grid, 404 if not found/not owned, 400 if malformed UUID

### DELETE /api/grids/[id] — Delete Grid
- **Ownership:** 404 if not owned
- **Cascade:** sets `deletedAt` on grid, all rows, all cells, all completions in a single ACID transaction
- **Response:** 204 no content, 404 if not found/not owned/already deleted, 400 if malformed UUID

## MCV Layer Structure

### Model (`src/models/grid.ts`)
- `createGrid(userId, data)` — validates name, creates grid, returns it
- `listGrids(userId)` — returns user's grids sorted by updatedAt desc
- `getGridById(gridId, userId)` — returns full nested grid or null
- `deleteGrid(gridId, userId)` — cascading soft delete in transaction, returns boolean
- All business rules live here: name validation, ownership checks, cascade logic

### Controller (`src/controllers/grid.ts`)
- One function per endpoint: `createGrid`, `listGrids`, `getGrid`, `deleteGrid`
- Parses request body/params, calls `getCurrentUserId()`, delegates to model
- Translates model results to HTTP responses (status codes, error shapes)
- Never touches Prisma directly

### View (`src/views/grid.ts`)
- Pure transform functions: `formatGrid`, `formatGridDetail`, `formatGridList`
- Strips internal fields (no `deletedAt`, no `userId` in response)
- Formats timestamps as ISO 8601 strings
- Rounds any floating-point tempo values to integers

### API Routes
- `src/app/api/grids/route.ts` — POST and GET collection
- `src/app/api/grids/[id]/route.ts` — GET and DELETE single resource
- Thin wrappers: import controller, call it, return response

## Error Response Shape

Consistent error format across all endpoints:

```json
{
  "error": {
    "message": "Human-readable error description",
    "code": "VALIDATION_ERROR"
  }
}
```

Error codes: `VALIDATION_ERROR`, `NOT_FOUND`, `INTERNAL_ERROR`
