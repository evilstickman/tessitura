# M1.1: Project Bootstrap — Design

## Goal
Set up the Next.js project with all tooling, testing, CI/CD, and environment separation. At completion, `npm run build && npm run test` passes on an empty scaffold.

## Legacy Code Handling
- Create a `legacy/django` branch from current `main` to preserve the Django codebase
- Remove all Django files from `main` (keep `docs/` directory)
- Git history is fully preserved

## Project Structure (MCV + Director Pattern)

```
tessitura/
├── src/
│   ├── models/               # M: Domain entities, business rules, Prisma queries
│   ├── controllers/          # C: Request handling, validation, orchestration
│   ├── views/                # V: Response serialization, API response formatting
│   ├── app/                  # Next.js App Router
│   │   ├── api/              # API route files (thin → delegate to controllers)
│   │   ├── layout.tsx        # Root layout
│   │   └── page.tsx          # Landing page
│   ├── components/           # Frontend components
│   │   ├── presentation/     # Dumb components (render props)
│   │   └── directors/        # Smart components (state, data fetching)
│   └── lib/                  # Shared utilities (not business logic)
│       ├── db.ts             # Prisma client singleton
│       └── env.ts            # Environment variable validation
├── prisma/
│   ├── schema.prisma         # Database schema (empty scaffold)
│   └── migrations/           # Prisma migrations
├── tests/
│   ├── unit/                 # Vitest unit tests
│   ├── integration/          # API endpoint tests (Vitest + test DB)
│   └── e2e/                  # Playwright e2e tests
├── docs/
│   ├── PRODUCT_SPEC.md       # Already exists
│   └── plans/                # Design documents
├── .github/
│   └── workflows/
│       └── ci.yml            # Lint, type-check, test, build
├── .env.example              # Environment variable template
├── .env.test                 # Test environment (committed, no secrets)
├── CLAUDE.md                 # Agentic documentation
├── package.json
├── tsconfig.json
├── tailwind.config.ts
├── vitest.config.ts
├── playwright.config.ts
└── next.config.ts
```

## Layer Boundaries

| Layer | Directory | Allowed Dependencies | Never |
|-------|-----------|---------------------|-------|
| Model | `src/models/` | Prisma client, other models | Controllers, views, components |
| Controller | `src/controllers/` | Models, views | Prisma directly, components |
| View | `src/views/` | None (pure transforms) | Models, controllers, Prisma |
| API Route | `src/app/api/` | Controllers only | Models, Prisma, views directly |
| Director | `src/components/directors/` | API calls (fetch/TanStack Query) | Models, controllers, Prisma |
| Presentation | `src/components/presentation/` | React props only | Everything else |

## Environment Configuration

Three environments via `.env` files:

| Environment | Database | Purpose |
|-------------|----------|---------|
| `test` | Fresh DB per suite run, torn down after | Automated tests |
| `dev` | Local persistent DB | Development |
| `prod` | DigitalOcean PostgreSQL | Production |

Environment variables validated at startup via `src/lib/env.ts` — app refuses to start if required vars are missing.

## CI/CD Pipeline (GitHub Actions)

Triggered on every push and PR:
1. Install dependencies
2. Lint (ESLint — zero errors, zero warnings)
3. Type-check (tsc --noEmit — zero errors)
4. Unit + integration tests (Vitest — 95% coverage gate)
5. Build (next build — must succeed)
6. E2E tests (Playwright — all pass)

On merge to main: deploy to DigitalOcean (automated).

## Completion Criteria

- `npm run dev` starts the application
- `npm run build` produces a production build
- `npm run test` runs Vitest (empty test passes)
- `npm run test:e2e` runs Playwright (empty test passes)
- `npm run lint` passes with zero errors
- `npx tsc --noEmit` passes
- CI pipeline runs and passes on GitHub Actions
- Prisma connects to local PostgreSQL
- Three environments (test/dev/prod) are configured and documented
- CLAUDE.md exists with project conventions
