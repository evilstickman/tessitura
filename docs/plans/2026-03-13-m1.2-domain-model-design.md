# M1.2: Domain Model — Design

## Goal
Define the Prisma schema for V1 entities (User, PracticeGrid, PracticeRow, PracticeCell, PracticeCellCompletion) with soft delete, proper constraints, and unit/integration tests.

## Scope
V1 fields only. V2+ fields (grid_type, archived, xp, streaks, subscriptions, etc.) will be added via migrations in later milestones. This follows YAGNI.

## Entities

### User
- id (UUID), email (unique), password_hash, display_name, instruments (text[])
- email_verified (bool, default false), timezone (string, default "UTC")
- default_fade_enabled (bool, default true)
- freshness_reset_strategy (enum: FULL/HALVE, default FULL)
- created_at, updated_at, deleted_at

### PracticeGrid
- id (UUID), user_id (FK→User), name (string, required), notes (text, nullable)
- fade_enabled (bool, defaults from user preference)
- created_at, updated_at, deleted_at

### PracticeRow
- id (UUID), practice_grid_id (FK→PracticeGrid)
- sort_order (int), song_title (nullable), composer (nullable), part (nullable)
- start_measure (string), end_measure (string)
- target_tempo (int), steps (int)
- priority (enum: CRITICAL/HIGH/MEDIUM/LOW, default MEDIUM)
- created_at, updated_at, deleted_at

### PracticeCell
- id (UUID), practice_row_id (FK→PracticeRow)
- step_number (int), target_tempo_percentage (float)
- freshness_interval_days (int, default 1)
- created_at, updated_at, deleted_at

### PracticeCellCompletion
- id (UUID), practice_cell_id (FK→PracticeCell)
- completion_date (date)
- created_at, deleted_at
- UNIQUE(practice_cell_id, completion_date) — scoped to non-deleted

## Key Design Decisions
- UUIDs for all primary keys
- Soft delete (deleted_at) on every entity
- Prisma client extension for automatic soft-delete filtering
- Cascading soft deletes via application logic (not DB cascade)
- Enums defined in Prisma: FreshnessResetStrategy, Priority

## Testing
- Unit tests: enum values, default values, field constraints
- Integration tests: CRUD operations, FK relationships, unique constraints, soft delete behavior, cascading soft delete
