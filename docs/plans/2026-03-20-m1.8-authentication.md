# M1.8: Authentication — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace placeholder auth with Google OAuth via Auth.js v5. Zero controller changes — only `getCurrentUserId()` internals change.

**Architecture:** Auth.js v5 with Google provider + Prisma adapter. JWT sessions (stateless). Middleware redirects unauthenticated users to sign-in page. `getCurrentUserId()` reads JWT via `auth()` helper in production, falls back to seed user in dev/test.

**Tech Stack:** Auth.js v5 (`next-auth@beta`), `@auth/prisma-adapter`, Next.js 16 middleware

**Spec:** `docs/specs/2026-03-20-m1.8-authentication-design.md`

```sql
-- Data-preserving: no data loss, existing users keep working

-- Step 1: Make password_hash nullable (OAuth users have no password)

ALTER TABLE "users"
  ALTER COLUMN "email_verified" DROP DEFAULT,
  ALTER COLUMN "email_verified" TYPE timestamptz
    USING CASE WHEN "email_verified" THEN NOW() ELSE NULL END;

-- Step 3: Make display_name nullable (Google may not provide a name)
ALTER TABLE "users" ALTER COLUMN "display_name" DROP NOT NULL;

-- Step 4: Add image column

-- Step 5: Set default for instruments (for Auth.js auto-created users)

-- Step 6: Create accounts table
CREATE TABLE "accounts" (
    "id" UUID NOT NULL DEFAULT gen_random_uuid(),
    "user_id" UUID NOT NULL,
    "type" TEXT NOT NULL,
    "provider" TEXT NOT NULL,
    "provider_account_id" TEXT NOT NULL,

    "token_type" TEXT,
    "scope" TEXT,
    "id_token" TEXT,
    "session_state" TEXT,

);

CREATE UNIQUE INDEX "accounts_provider_provider_account_id_key"
  ON "accounts"("provider", "provider_account_id");
```

```ts
import NextAuth from 'next-auth';
import Google from 'next-auth/providers/google';
import { PrismaAdapter } from '@auth/prisma-adapter';
import { prisma } from '@/lib/db';

export const { handlers, auth, signIn, signOut } = NextAuth({
  adapter: PrismaAdapter(prisma),
  providers: [Google],
  session: { strategy: 'jwt' },
  pages: { signIn: '/auth/signin' },
  callbacks: {
    jwt({ token, user }) {
      if (user?.id) token.sub = user.id;
      return token;
    },
    session({ session, token }) {
      if (token.sub && session.user) {
        session.user.id = token.sub;
      }
      return session;
    },
  },
});
```

- [ ] **Step 2: Implement the new getCurrentUserId**

Replace `src/lib/auth.ts`:

```ts
import { AuthenticationError } from '@/lib/errors';

let seedUserId: string | null = null;

/**
 * Resets the cached seed user ID. Used by test setup to prevent stale cache.
 */
export function resetAuthCache(): void {
  seedUserId = null;
}

/**
 * Returns the current user's ID.
 *
 * Production (AUTH_SECRET set): reads JWT via Auth.js auth() helper.
 * Development (no AUTH_SECRET): falls back to dev seed user.
 *
 * Throws AuthenticationError on failure → controllers return 401.
 * This contract is permanent across all environments.
 */
export async function getCurrentUserId(): Promise<string> {
  // Production mode: use Auth.js session
  if (process.env.AUTH_SECRET) {
    const { auth } = await import('@/lib/auth.config');
    const session = await auth();

    if (!session?.user?.id) {
      throw new AuthenticationError('Not authenticated');
    }

    return session.user.id;
  }

  // Dev/test fallback: seed user
  if (seedUserId) return seedUserId;

  const { prisma } = await import('@/lib/db');
  const user = await prisma.user.findFirst({
    where: { email: 'dev-placeholder@tessitura.local' },
  });

  if (!user) {
    throw new AuthenticationError(
      'Not authenticated. Dev seed user not found — run `npx prisma db seed` to create it.',
    );
  }

  seedUserId = user.id;
  return seedUserId;
}
```

```ts
import { auth } from '@/lib/auth.config';

export default auth((req) => {
  if (!req.auth && req.nextUrl.pathname !== '/auth/signin') {
    const signInUrl = new URL('/auth/signin', req.nextUrl.origin);
    return Response.redirect(signInUrl);
  }
});

export const config = {
  matcher: ['/((?!api/auth|_next/static|_next/image|favicon\\.ico|auth/).*)'],
};
```

This protects all routes except `/api/auth/*` (NextAuth routes), `/_next/*` (static assets), `/favicon.ico`, and `/auth/*` (sign-in page).
