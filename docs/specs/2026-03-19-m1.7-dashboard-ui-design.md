# M1.7: Dashboard UI — Design Spec

## Scope

4-quadrant dashboard replacing the landing page. Displays alerts, grid list with stats, statistics, and practice focus suggestions. Includes basic grid creation (name + notes).

**Use cases covered:** UC-1.5 (Dashboard Layout), UC-1.5a (Alerts), UC-1.5b (My Grids), UC-1.5c (Statistics), UC-1.5d (Practice Focus)

**Explicitly NOT in scope:** Streak calculation (placeholder), "See all alerts" page, grid editing/deletion from dashboard, row creation from dashboard, real-time updates, social notifications (V5+), practice time charts (V2+), achievement showcase (V3+).

## Route Structure

- `/` — Dashboard. Replaces the current landing page. Renders DashboardDirector.
- `/grids` — Full grid list page (new). Moves GridListDirector here.
- `/grids/[id]` — Grid detail page (existing from M1.6, unchanged).

## Backend Change: Enriched Grid List

`GET /api/grids` accepts an optional `?detail=true` query parameter.

**Default (no param — lightweight, unchanged):**
```json
[{ "id": "...", "name": "...", "notes": "...", "fadeEnabled": true, "createdAt": "...", "updatedAt": "..." }]
```

**With `?detail=true` (enriched):**
```json
[{
  "id": "...", "name": "...", "notes": "...", "fadeEnabled": true,
  "createdAt": "...", "updatedAt": "...",
  "completionPercentage": 67,
  "freshnessSummary": { "fresh": 5, "aging": 2, "stale": 1, "decayed": 0, "incomplete": 7 },
  "rows": [{
    "id": "...", "piece": { "title": "...", "composer": "..." }, "passageLabel": "...",
    "startMeasure": 1, "endMeasure": 16, "priority": "HIGH",
    "completionPercentage": 50,
    "freshnessSummary": { "fresh": 3, "aging": 0, "stale": 1, "decayed": 0, "incomplete": 1 }
  }]
}]
```

The detail response includes row-level summary data (needed for Practice Focus ranking) but NOT individual cells — cells are only needed on the grid detail page.

**`formatGridSummaryList` row output shape (explicit):** Each row in the summary contains only: `id`, `piece` (title + composer), `passageLabel`, `startMeasure`, `endMeasure`, `priority`, `completionPercentage`, `freshnessSummary`. Fields omitted from the full `formatRow`: `sortOrder`, `targetTempo`, `steps`, `createdAt`, `updatedAt`, `cells`.

**Implementation:** The controller reads `detail` from the URL search params and captures `const now = new Date()`. When `detail=true`, it calls a new `listGridsWithDetail(userId)` model function. This function performs a single Prisma query that fetches all grids with their nested rows, cells, and completions (the same includes as `getGridById` but for all grids at once, filtered by `userId` and `deletedAt: null`). The controller then passes the result to a new `formatGridSummaryList(grids, now)` view function that computes freshness stats per grid and per row (reusing the existing freshness calculation functions, threading each grid's `fadeEnabled` value through to `calculateCompletionPercentage`) and strips cell-level data from the response.

The model returns raw data; the controller owns `now` — matching the established pattern from `getGrid`/`formatGridDetail`. This avoids N+1 queries — one database call regardless of grid count.

The existing `GET /api/grids` (no param) response shape stays unchanged. However, the current `listGrids` model has a pre-existing bug: it queries `where: { userId }` without `deletedAt: null`, returning soft-deleted grids. **Fix this as part of M1.7** — add `deletedAt: null` to the `listGrids` query. This affects both the lightweight and detail paths.

## Component Architecture

```
src/app/page.tsx                              ← Dashboard page, renders DashboardDirector
  └─ DashboardDirector                        ← Fetches grids?detail=true, derives pane data
       └─ DashboardLayout                     ← 2x2 grid (desktop) / stacked (mobile)
            ├─ AlertsPane                     ← Greeting + actionable alerts
            ├─ MyGridsPane                    ← Grid cards + GridCreateForm + "See all"
            │    └─ GridCreateForm            ← Inline name+notes form
            ├─ StatisticsPane                 ← Avg completion %, total cells, streak placeholder
            └─ PracticeFocusPane              ← Top 5 rows by priority + decay

src/app/grids/page.tsx                        ← Full grid list, renders GridListDirector
```

### Layer Rules (per CLAUDE.md)

- **DashboardDirector**: owns TanStack Query state. Fetches `GET /api/grids?detail=true`. Derives alert items, statistics, and practice focus suggestions from the response. Creates grid mutation for the creation form. Passes all computed data as props.
- **All panes + DashboardLayout**: pure presentation, props only.
- **GridCreateForm**: presentation component with controlled inputs (local state for form fields only). Receives `onSubmit(name, notes)` and `onCancel` callbacks from the director via MyGridsPane.

## Data Derivation (in DashboardDirector)

All dashboard data is derived from the single `GET /api/grids?detail=true` response. No additional endpoints.

### Alerts

Derived from grid-level `freshnessSummary`. Two alert types per UC-1.5a:

**Resume prompts** (internal state: `freshnessSummary.decayed > 0`):
- For each grid with decayed cells:
  - Display copy: "Resume {grid name} — {N} cells need re-practice"
  - Border color: `#ef4444` (red). Link: `/grids/{id}`

**Decay warnings** (internal state: `freshnessSummary.stale > 0`):
- For each grid with stale cells (completed but overdue, will decay if not practiced soon):
  - Display copy: "{grid name} — {N} cells losing freshness"
  - Border color: `#fbbf24` (amber). Link: `/grids/{id}`

- Sort: resume prompts first (red), then decay warnings (amber). Within each group, sort by count descending.
- Max 5 alerts shown.
- "See all →" link shown when more than 5 alerts exist. Links to `/grids` (no dedicated alerts page in V1 — the full grid list serves as the expanded view). Per UC-1.5a AC.

### Statistics

- **Avg completion**: mean of `completionPercentage` across all grids, displayed as integer %. If zero grids, display 0%.
- **Cells completed**: sum of `freshnessSummary.fresh + aging + stale + decayed` across all grids. This counts cells "ever completed" (including decayed). Note: this differs from `completionPercentage`, which counts "currently completed" (excludes decayed when fade is on). The distinction is intentional — the stat tracks lifetime effort, the percentage tracks current state.
- **Streak**: "—" placeholder with "Coming soon" label

### Practice Focus

- Collect all rows across all grids (available in the detail response)
- Each row has `freshnessSummary` with per-row stale/decayed counts
- Ranking: primary sort by priority (CRITICAL=0 > HIGH=1 > MEDIUM=2 > LOW=3), secondary sort by `stale + decayed` count descending (most urgent first)
- Only include rows where `stale + decayed > 0` (rows that need practice)
- Take top 5
- Each suggestion shows: piece/passage label (same logic as GridRow), grid name, priority badge, "{N} cells need practice"
- Each suggestion links to `/grids/{gridId}`
- If no rows need practice: show top 5 rows sorted by priority regardless (everything is fresh — suggest what to practice next). Subtitle text: "All fresh — keep it up!" instead of "{N} cells need practice"

## Visual Design

### Layout

Card-style quadrants on dark background:
- Page background: `#111827`
- Each quadrant: `background: #1f2937`, `border-radius: 6px`, `padding: 16px`
- Desktop (768px+): CSS grid `grid-template-columns: 1fr 1fr`, `gap: 12px`, page padding `12px`
- Mobile (<768px): single column, quadrants stack: Alerts → My Grids → Statistics → Practice Focus

### Quadrant Headers

Each pane has a header: uppercase, `11px`, `#6b7280`, `letter-spacing: 0.05em`. Headers read: "Alerts", "My Grids", "Statistics", "Practice Focus".

### AlertsPane

- Greeting: "Welcome back, Dev User" (`15px`, `#f9fafb`, semibold). Display name hardcoded until M1.8.
- Alert items: `#111827` background, `border-radius: 4px`, `padding: 8px 10px`. Left border `3px solid` — red (`#ef4444`) for decayed alerts, amber (`#fbbf24`) for stale-only. Arrow `→` on right.
- Empty state (no grids): "Create your first practice grid to get started"

### MyGridsPane

- Header row: "My Grids" label + green "New Grid" button (`#10b981` background, white text)
- Grid cards: `#111827` background, `border-radius: 4px`. Each shows: grid name, mini progress bar (60px wide, 4px tall, `#374151` track / `#10b981` fill), percentage label. Below the name: `updatedAt` as "Updated M/DD" (`#6b7280`, `11px`) — this is a generic record-update timestamp, not a guaranteed practice-activity time, so the label is deliberately neutral. Also shows stale+decayed count if >0 as "{N} cells need attention" (`#fbbf24` amber, `11px`). Per UC-1.5b.
- Cards are clickable links to `/grids/{id}`
- Max 5 grids, sorted by `updatedAt` descending
- "See all →" link (right-aligned, `#6b7280`, `11px`) when more than 5 grids exist. Links to `/grids`.
- Empty state: "No practice grids yet." + New Grid button still visible

### GridCreateForm (inline, appears above grid list when triggered)

- Name input: text, required, placeholder "Grid name", `#111827` background, `#f9fafb` text, border `1px solid #374151`
- Notes input: text, optional, placeholder "Notes (optional)", same styling
- Buttons: "Create" (`#10b981` green) + "Cancel" (`#374151` gray), `11px`
- On submit: director calls `POST /api/grids`, invalidates `['grids']` query. Form collapses on success.
- On error: inline error text below form in `#ef4444`

### StatisticsPane

- Three stat columns in a `grid-template-columns: 1fr 1fr 1fr` layout
- Avg completion: large number (`24px`, `#10b981` green), label below (`11px`, `#6b7280`)
- Cells completed: large number (`24px`, `#f9fafb`), label below
- Streak: "—" in `#374151` (dim), label "Streak (soon)"
- Empty state (no grids): "Complete your first cell to start tracking!" centered

### PracticeFocusPane

- Suggestion items: `#111827` background, `border-radius: 4px`, `padding: 8px 10px`
- Priority badge: colored pill — CRITICAL: `#7f1d1d` bg / `#fca5a5` text, HIGH: `#78350f` bg / `#fbbf24` text, MEDIUM: `#374151` bg / `#9ca3af` text, LOW: same as MEDIUM
- Each item: priority badge, piece/passage label (`#f9fafb`, `12px`), subtitle "{grid name} · {N} cells need practice" (`#9ca3af`, `11px`)
- Items are clickable links to `/grids/{gridId}`
- Max 5 shown
- Empty state: "Start practicing to see personalized suggestions here"

## Error Handling

| Scenario | Behavior |
|---|---|
| Dashboard loading | "Loading..." text centered on page |
| Query error (network/500) | "Failed to load dashboard." centered on page |
| Auth error (401) | Show "Authentication required" message (no redirect). Both DashboardDirector and GridListDirector display this inline instead of redirecting — avoids the redirect loop between `/` and `/grids`. When M1.8 (NextAuth) lands, both redirect to login instead. Also update the existing GridListDirector and GridViewDirector to match this pattern (replace `window.location.href = '/'` with inline error display). |
| Grid creation 400 (validation) | Inline error text below form |
| Grid creation network error | Inline error text "Failed to create grid" |
