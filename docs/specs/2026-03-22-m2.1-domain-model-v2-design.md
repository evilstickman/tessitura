# M2.1: Domain Model V2 — Design Spec

## Scope

Extend the Prisma schema with 3 new entities (PracticeSession, PracticeGoal, LibraryTemplate), add 3 fields to PracticeGrid (gridType, archived, sourceTemplateId), and build full CRUD API endpoints for all new entities following the established MCV pattern.

**Use cases covered:** UC-2.1 (Multiple Grids — archived filter), UC-2.2 (Practice Time — session model), UC-2.5 (Goal Setting — goal model), UC-2.7 (Technique Grid — gridType enum), UC-2.8 (Library — template model + clone)

**Explicitly NOT in scope:** Session inference from cell completions (M2.3), goal progress computation (M2.5 UI), template admin CRUD (UC-2.11), community stats aggregation (UC-2.10), billing/tier enforcement, practice feed algorithm (UC-2.6).

## Schema Changes

### New Enums

```prisma
enum GridType {
  REPERTOIRE
  TECHNIQUE
}

enum SessionSource {
  MANUAL
  INFERRED
}

enum GoalType {
  DAILY_MINUTES
  WEEKLY_MINUTES
  WEEKLY_SESSIONS
  MONTHLY_GRIDS
}

enum TierRequired {
  FREE
  PRO
}
```

### PracticeGrid Additions

```prisma
model PracticeGrid {
  // Existing fields unchanged, add:
  gridType         GridType          @default(REPERTOIRE) @map("grid_type")
  archived         Boolean           @default(false)
  sourceTemplateId String?           @map("source_template_id") @db.Uuid

  sourceTemplate    LibraryTemplate?  @relation(fields: [sourceTemplateId], references: [id])
  practiceSessions  PracticeSession[]

  // NEW indexes
  @@index([userId, archived])
  @@index([sourceTemplateId])
}
```

- `gridType`: affects UI labels (repertoire vs technique), not behavior. Default REPERTOIRE.
- `archived`: controls list visibility only — archived grids are excluded from all list/detail endpoints by default (including `GET /api/grids`, `GET /api/grids?detail=true`, and dashboard data). Archived grids remain fully accessible via `GET /api/grids/{id}` (direct access by ID). All data, rows, cells, and completions are preserved unchanged. Cloned grids start with `archived = false`. Archiving does not affect freshness calculations, completion percentages, or any other behavior — it is purely a list filter.
- `sourceTemplateId`: FK to LibraryTemplate. Null for user-created grids. Set once during clone, immutable.

### New Models

**PracticeSession** — tracks manual practice time entries.

```prisma
model PracticeSession {
  id              String        @id @default(uuid()) @db.Uuid
  userId          String        @map("user_id") @db.Uuid
  practiceGridId  String?       @map("practice_grid_id") @db.Uuid
  sessionDate     DateTime      @map("session_date") @db.Date
  durationMinutes Int           @map("duration_minutes")
  notes           String?
  source          SessionSource @default(MANUAL)
  createdAt       DateTime      @default(now()) @map("created_at") @db.Timestamptz
  deletedAt       DateTime?     @map("deleted_at") @db.Timestamptz

  user         User          @relation(fields: [userId], references: [id])
  practiceGrid PracticeGrid? @relation(fields: [practiceGridId], references: [id])

  @@index([userId, sessionDate])
  @@map("practice_sessions")
}
```

Sessions are immutable records. No `updatedAt` — if logged wrong, delete and re-create. Manual only in M2.1 (no inference from completions).

**sessionDate semantics:** Stored as `@db.Date` (date-only, no time component). The API accepts and returns YYYY-MM-DD strings only — same pattern as `PracticeCellCompletion.completionDate`. No timezone conversion. String manipulation only (split, parse, format). The `?from` and `?to` date range filters are inclusive and also use YYYY-MM-DD strings. No `Date` object parsing anywhere in the stack — this is a calendar date, not a moment in time.

**PracticeGoal** — user-set practice targets.

```prisma
model PracticeGoal {
  id          String   @id @default(uuid()) @db.Uuid
  userId      String   @map("user_id") @db.Uuid
  goalType    GoalType @map("goal_type")
  targetValue Int      @map("target_value")
  active      Boolean  @default(true)
  createdAt   DateTime @default(now()) @map("created_at") @db.Timestamptz
  updatedAt   DateTime @updatedAt @map("updated_at") @db.Timestamptz
  deletedAt   DateTime? @map("deleted_at") @db.Timestamptz

  user User @relation(fields: [userId], references: [id])

  @@index([userId])
  @@map("practice_goals")
}
```

`goalType` is immutable after creation — prevents changing measurement unit. `targetValue` and `active` are updatable. Progress is computed on read (not stored), deferred to M2.5 UI.

**Multiple goals:** Only one active goal per goalType is allowed. Creating a new goal with the same goalType as an existing active goal returns 409 CONFLICT. The user must deactivate the existing goal first (PUT with `active: false`) before creating a new one. Inactive goals of any type are unlimited (historical records).

**LibraryTemplate** — curated practice grid blueprints.

```prisma
model LibraryTemplate {
  id             String       @id @default(uuid()) @db.Uuid
  title          String
  author         String
  collection     String
  description    String?
  instrumentTags String[]     @default([]) @map("instrument_tags")
  gridType       GridType     @default(REPERTOIRE) @map("grid_type")
  tierRequired   TierRequired @default(FREE) @map("tier_required")
  gridData       Json         @map("grid_data")
  active         Boolean      @default(true)
  createdAt      DateTime     @default(now()) @map("created_at") @db.Timestamptz
  updatedAt      DateTime     @updatedAt @map("updated_at") @db.Timestamptz
  deletedAt      DateTime?    @map("deleted_at") @db.Timestamptz

  clonedGrids    PracticeGrid[]

  @@map("library_templates")
}
```

`gridData` is a JSON blob: `{ rows: [{ passageLabel?, startMeasure, endMeasure, targetTempo, steps }] }`. Templates are read-only for users. Admin CRUD comes in a later milestone. Seeded via seed script for dev/testing.

**gridData → PracticeRow mapping during clone:** Each row in gridData maps to a PracticeRow with `pieceId = null` (template rows are not linked to user pieces), `passageLabel` from gridData (or null), and the numeric fields directly. The template's `title` and `author` are used for the grid name, not for individual rows.

`tierRequired` is stored but NOT enforced in M2.1 — all templates are accessible. Enforcement comes with billing (V4+).

## API Endpoints

### PracticeGrid Updates

**`GET /api/grids`** — updated:
- Add `?archived` filter: `?archived=false` (default, active grids only), `?archived=true` (archived grids only), `?archived=all` (both). Omitting the param is equivalent to `?archived=false`.
- Composes with existing `?detail=true` param — both filters apply simultaneously.
- Both `listGrids()` and `listGridsWithDetail()` must accept the archived filter.

**`PUT /api/grids/{id}`** — new:
- Body: `{ name?, notes?, gridType?, archived?, fadeEnabled? }`
- Validates: name non-empty if provided, gridType is valid enum
- Returns updated grid via `formatGrid()`
- Cannot change `sourceTemplateId` (immutable)
- Subsumes the existing `PUT /api/grids/{id}/fade` endpoint's functionality. Both endpoints call the same underlying `updateGrid()` model function — the fade endpoint becomes a thin wrapper that calls `updateGrid(gridId, userId, { fadeEnabled })`. This ensures a single code path for all grid updates.

### PracticeSession CRUD

**`POST /api/sessions`** — create manual session
- Body: `{ practiceGridId?, sessionDate, durationMinutes, notes? }`
- `sessionDate`: YYYY-MM-DD string
- `durationMinutes`: positive integer
- `practiceGridId`: optional, validated as UUID and owned by user if provided
- `source` auto-set to `MANUAL`
- Returns 201 with formatted session

**`GET /api/sessions`** — list user's sessions
- Sorted by `sessionDate` desc, then `createdAt` desc
- Optional `?from=YYYY-MM-DD&to=YYYY-MM-DD` date range filter
- Returns array of formatted sessions

**`GET /api/sessions/{id}`** — get single session
- Scoped to user (ownership check)
- Returns formatted session or 404

**`DELETE /api/sessions/{id}`** — soft delete
- Scoped to user
- Returns 204

No PUT — sessions are immutable practice records.

### PracticeGoal CRUD

**`POST /api/goals`** — create goal
- Body: `{ goalType, targetValue }`
- `goalType`: valid GoalType enum value
- `targetValue`: positive integer
- Returns 201 with formatted goal

**`GET /api/goals`** — list user's goals
- Default: active goals only
- `?all=true`: includes inactive goals
- Sorted by `createdAt` desc

**`PUT /api/goals/{id}`** — update goal
- Body: `{ targetValue?, active? }`
- Cannot change `goalType` (returns 400 if attempted)
- Returns updated goal

**`DELETE /api/goals/{id}`** — soft delete
- Returns 204

### LibraryTemplate (read-only for users)

**`GET /api/templates`** — list active templates
- Optional `?instrument=trumpet` filter — case-insensitive, matches if the value is contained in any element of the `instrumentTags` array (Prisma's `has` filter with lowercased comparison). Single instrument per query.
- Returns array of formatted templates (without gridData — list is summary only)

**`GET /api/templates/{id}`** — get template detail
- Returns template with gridData included

**`POST /api/templates/{id}/clone`** — clone template to user's grids
- Validates template exists and is active
- Reads `gridData` JSON, creates:
  - PracticeGrid (name = template title, gridType = template.gridType, sourceTemplateId = template.id)
  - PracticeRows from each row in gridData
  - PracticeCells from each row's steps (using existing cell generation formula: `0.4 + 0.6 * step / (total - 1)`)
- Returns 201 with the newly created grid detail (via formatGridDetail — includes rows and cells so the client can display the new grid immediately)

## View Functions

**`formatSession(session)`** — returns: id, practiceGridId, sessionDate (YYYY-MM-DD), durationMinutes, notes, source, createdAt. Strips userId, deletedAt.

**`formatGoal(goal)`** — returns: id, goalType, targetValue, active, createdAt, updatedAt. Strips userId, deletedAt.

**`formatTemplate(template)`** — returns: id, title, author, collection, description, instrumentTags, tierRequired, active, createdAt, updatedAt. Strips deletedAt. Does NOT include gridData (list view).

**`formatTemplateDetail(template)`** — same as formatTemplate + gridData.

**`formatGrid(grid)`** — updated to include gridType, archived, sourceTemplateId.

## Validation Rules

**PracticeSession:**
- `durationMinutes` ≥ 1, ≤ 1440 (24 hours max)
- `sessionDate` must be a valid YYYY-MM-DD string
- `practiceGridId` if provided: valid UUID, grid exists and owned by user
- `notes` max 2000 characters

**PracticeGoal:**
- `targetValue` ≥ 1
- `goalType` must be valid enum
- Update: `goalType` cannot be changed (400 if present in body)
- `targetValue` ≤ 10000 (reasonable upper bound)

**PracticeGrid update:**
- `name` non-empty if provided (same rules as create)
- `gridType` must be valid enum if provided
- `sourceTemplateId` cannot be set via update (immutable)

**LibraryTemplate clone:**
- Template must exist and be active
- `gridData` must have valid structure: `{ rows: [...] }` with at least 1 row
- Each row validates with the same bounds as PracticeRow creation: `startMeasure` ≥ 1, ≤ 99999; `endMeasure` ≥ 1, ≤ 99999; `targetTempo` ≥ 1, ≤ 999; `steps` ≥ 1, ≤ 50. Use the existing `validateRowInput` from `src/models/row.ts` to validate each row.
- Note: gridData is also validated in the seed script when seeding templates. Since admin CRUD is out of scope, the only way templates enter the system is via seed — the seed must validate structure before inserting.

**Clone is transactional:** The entire clone operation (create grid + create all rows + create all cells) runs inside a single `prisma.$transaction`. If any row or cell creation fails, the entire clone rolls back — no partial grids.

## Error Handling

Same patterns as V1:
- 400 for validation errors (VALIDATION_ERROR)
- 401 for auth failures (AUTHENTICATION_ERROR)
- 404 for not found / not owned (NOT_FOUND)
- 409 for conflicts (CONFLICT)
- 500 for unexpected errors (INTERNAL_ERROR)

Goal-specific: 400 if PUT body contains `goalType` ("Goal type cannot be changed after creation").

**Unit tests:**
- `tests/unit/models/session.test.ts`: validation (duration bounds, date parsing)

- `tests/unit/models/template.test.ts`: gridData validation
