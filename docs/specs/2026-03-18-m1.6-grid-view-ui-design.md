# M1.6: Grid View UI — Design Spec

## Scope

View-only grid detail page with cell interactions. No grid/row creation or editing forms.

**Use cases covered:** UC-1.6 (Complete Cell), UC-1.7a (Undo Completion), UC-1.11 (View Priority), UC-1.12 (View Practice Grid)

**Explicitly NOT in scope:** UC-1.7b Reset (API exists, no UI yet), grid/row CRUD forms, touch undo gesture, dark/light theme toggle, animations.

## Route Structure

- `/` — Landing page. Fetches `GET /api/grids`, renders list of grid links (name + completion %).
- `/grids/[id]` — Grid detail page. Fetches full grid data, renders the practice grid table with cell interactions.

## Component Architecture

```
src/app/grids/[id]/page.tsx              ← Next.js page, extracts gridId
  └─ GridViewDirector                    ← TanStack Query: useQuery + useMutation
       ├─ GridHeader                     ← name, notes, progress, summary, fade toggle
       └─ GridTable                      ← <table> wrapper, overflow-x scroll
            └─ GridRow (×N)              ← <tr>: passage header cell + practice cells
                 └─ PracticeCell (×N)    ← <td>: color, content, click handlers
```

### Layer Rules (per CLAUDE.md)

- **Director** (`GridViewDirector`): owns TanStack Query state, creates mutation callbacks, passes data + callbacks as props. Never renders domain-specific UI itself.
- **Presentation** (all others): receive props only. No fetch, no side effects. Exception: `GridHeader` has local `useState` for collapse toggle.
- **Page** (`grids/[id]/page.tsx`): thin wrapper, renders director with gridId prop.

## State Management

**TanStack Query** (add `@tanstack/react-query` dependency).

- `QueryClientProvider` wraps the app via a client component at `src/lib/query-client.tsx`.
- Grid detail query key: `['grid', gridId]`.
- `staleTime: 0` — freshness states change with time, always refetch after focus/mutation.

**Mutation pattern (refetch-after-mutate, no optimistic updates):**

```
onComplete(cellId) →
  POST /api/grids/{id}/rows/{rowId}/cells/{cellId}/complete →
  on success: invalidateQueries(['grid', gridId]) →
  automatic refetch → UI updates from server truth
```

Same pattern for undo (`POST .../undo`) and fade toggle (`PUT /api/grids/{id}/fade`).

**Why no optimistic updates:** Completing one cell can change the shielding state of every other cell in the row (cascading fade). The server response is the only correct source of truth for the full row state after a mutation.

## Visual Design

### Theme

Dark theme only. Background: `#111827`. Table cell borders: `1px solid #374151`, `border-collapse: collapse`. No gaps, no rounding. Flush spreadsheet style matching the original Django app.

### Cell Color Mapping (Green Fade Palette)

The "green drains away" metaphor — completed cells fill with green, then the green literally fades as freshness decays. Decayed cells return to looking empty.

| `freshnessState` | Background | Text Content | Text Color |
|---|---|---|---|
| `incomplete` | `#1f2937` | BPM (e.g. "106") | `#9ca3af` |
| `fresh` | `#10b981` | Date (e.g. "3-15") | `#ffffff` |
| `aging` | `#6ee7b7` | Date | `#064e3b` |
| `stale` | `#a7f3d0` | Date | `#064e3b` |
| `decayed` | `#1f2937` | BPM | `#9ca3af` |

**Cell content rules:**
- Incomplete and decayed cells show `targetTempoBpm` from the API response.
- Fresh, aging, and stale cells show `lastCompletionDate` formatted as `M-DD` (strip leading zeros from the `YYYY-MM-DD` API string, drop the year).
- Cursor: `pointer` on all cells.

### Row Header (First Column)

Each row's first `<td>` contains:
- **Line 1:** Piece title or passage label (white, semibold). If both exist, show piece title. If neither, show "Untitled".
- **Line 2:** `mm. {start}–{end} · {priority}` (gray, 11px). Priority colored: CRITICAL=red, HIGH=amber, MEDIUM=gray, LOW=gray.

Min-width: 140px. Background: `#1f2937`.

### Grid Header

Two states, toggled by a chevron button:

**Expanded (default):**
- Row 1: Grid name (`<h1>`, 18px, semibold, white) + fade toggle (right-aligned)
- Row 2: Notes (12px, gray) — only rendered if notes exist
- Row 3: Progress bar (full-width, 8px tall, `#374151` track, `#10b981` fill) + percentage label
- Row 4: Freshness summary — colored dots with counts using cell palette colors

**Collapsed:**
- Single row: Grid name + progress bar + percentage + fade toggle
- Notes and freshness summary hidden
- Chevron icon rotates to indicate state

Collapse toggle is local `useState`, defaults to expanded, not persisted.

**Fade toggle:**
- Pill-shaped switch (36×20px), green when ON, `#374151` when OFF.
- Clicking calls mutation: `PUT /api/grids/{id}/fade` with `{ fadeEnabled: !current }`.
- On refetch, all cell states update (fade OFF → all completed cells show as `fresh`).

**Progress bar:**
- Uses `completionPercentage` from the API (server-computed).
- Displayed as integer (e.g. "67%").

## Interactions

**Left-click on any cell:** Calls `POST /api/grids/{id}/rows/{rowId}/cells/{cellId}/complete`. Grid refetches. Cell turns green (or stays green with updated date if re-completing a stale cell).

**Right-click on any cell:** Prevents browser context menu. Calls `POST /api/grids/{id}/rows/{rowId}/cells/{cellId}/undo`. Grid refetches. If no completions exist, API returns 409 — silently ignored.

**Reset:** Not exposed in M1.6 UI. API endpoint exists for future use.

## Responsive Layout

**Desktop (768px+):** Table renders naturally. Passage header column + cells in a single row. Horizontal scroll only if step count is very large (12+ steps).

**Mobile (320px+):**
- Header stacks: name above, progress bar below, fade toggle right-aligned.
- Table wrapped in `overflow-x: auto` — horizontal swipe to see more cells.
- Cells stay the same size (no shrinking).
- Left-click (tap) = complete. Right-click undo not available on touch — acknowledged limitation for M1.6.
- No sticky first column for M1.6.

**Rows with different step counts:** Produce a ragged right edge. Natural and expected — no padding cells added.

## Error Handling

| Scenario | Behavior |
|---|---|
| Grid loading | "Loading..." text (no skeleton for M1.6) |
| Grid not found (404) | "Grid not found" message |
| Cell mutation 409 (conflict) | Silently ignored — refetch shows correct state |
| Cell mutation 404 | Silently ignored — refetch shows correct state |
| Network error | TanStack Query default retry (3 attempts). If all fail, prior state remains. |

No toasts, alerts, or error banners for M1.6. The refetch-after-mutate pattern ensures the UI self-corrects.

## Testing Strategy

**Unit tests (Vitest + React Testing Library):**
- PracticeCell: renders correct background color and content for each of the 5 freshness states
- PracticeCell: fires onComplete on click, onUndo on right-click
- GridRow: renders passage header with piece/measures/priority, renders N cells
- GridHeader: renders name, progress bar, freshness summary; collapse toggle hides/shows details
- GridHeader: fade toggle calls onToggleFade

**Integration tests (Vitest):**
- GridViewDirector with mocked fetch: verify initial query, verify mutation triggers refetch

**E2E tests (Playwright):**
- Navigate to grid page, verify cells render with correct colors
- Click a cell, verify it turns green with completion date
- Right-click a completed cell, verify it reverts
- Toggle fade off, verify all completed cells show as fresh (same green)
- Screenshot each visual state for regression

**Coverage:** 95% minimum on all new code. The `src/components/**` exclusion in vitest.config.ts must be removed now that components exist.
