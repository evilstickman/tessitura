# M1.8: Authentication — Design Spec

## Scope

Google OAuth via NextAuth.js. Replaces the placeholder auth with real session management. Zero controller changes — only `getCurrentUserId()` internals are replaced.

**Use cases covered:** UC-1.1 (Registration — via Google OAuth auto-creation), UC-1.2 (Login/Logout — via NextAuth)

**Explicitly NOT in scope:** Apple OAuth (later milestone), email/password registration, email verification flow, forgot password, account lockout, profile/settings page, onboarding flow, force-logout / session revocation.

## Auth Strategy

- **Provider:** Google OAuth only (via NextAuth.js)
- **Session:** JWT (stateless, no database session table, no DB lookup per request)
- **Adapter:** `@auth/prisma-adapter` for user/account persistence
- **User creation:** Automatic on first Google sign-in (NextAuth handles via adapter)
- **Post-login:** Straight to dashboard — no onboarding

## Schema Changes

### New Models (NextAuth standard tables)

**Account** — stores OAuth provider data. NextAuth manages entirely.

```prisma
model Account {
  id                String  @id @default(uuid()) @db.Uuid
  userId            String  @map("user_id") @db.Uuid
  type              String
  provider          String
  providerAccountId String  @map("provider_account_id")
  refresh_token     String? @db.Text
  access_token      String? @db.Text
  expires_at        Int?
  token_type        String?
  scope             String?
  id_token          String? @db.Text
  session_state     String?

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([provider, providerAccountId])
  @@map("accounts")
}
```

**VerificationToken** — required by NextAuth adapter schema, unused for Google OAuth. Will be used when email/password is added.

```prisma
model VerificationToken {
  identifier String
  token      String   @unique
  expires    DateTime

  @@unique([identifier, token])
  @@map("verification_tokens")
}
```

### User Model Changes

```prisma
model User {
  // Existing fields unchanged except:
  passwordHash  String?  @map("password_hash")  // Was String, now nullable (OAuth users have no password)
  image         String?                          // NEW: Google profile picture URL

  // NEW relation:
  accounts      Account[]

  // All other fields and relations unchanged
}
```

**Migration:** Make `password_hash` nullable, add `image` column, create `accounts` and `verification_tokens` tables. Non-destructive — existing seed user keeps working.

## NextAuth Configuration

### Auth Options (`src/lib/auth-options.ts`)

Shared config used by both the route handler and `getToken()`:

- **Google provider:** `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET` from env
- **Prisma adapter:** connects to existing database via `src/lib/db.ts`
- **JWT session strategy:** `session: { strategy: 'jwt' }`
- **Callbacks:**
  - `jwt`: includes `user.id` as `token.sub` on sign-in
  - `session`: exposes `token.sub` as `session.user.id` for client-side access
- **Custom pages:** `signIn: '/auth/signin'`

### Route Handler (`src/app/api/auth/[...nextauth]/route.ts`)

Standard NextAuth catch-all:

```ts
import NextAuth from 'next-auth';
import { authOptions } from '@/lib/auth-options';

const handler = NextAuth(authOptions);
export { handler as GET, handler as POST };
```

### Environment Variables (new)

```
GOOGLE_CLIENT_ID=<from Google Cloud Console>
GOOGLE_CLIENT_SECRET=<from Google Cloud Console>
NEXTAUTH_SECRET=<random 32+ char string for JWT signing>
NEXTAUTH_URL=https://practicegrids.com
```

For local development without Google OAuth: omit these variables. `getCurrentUserId()` falls back to seed user behavior.

## Auth Integration — `getCurrentUserId()` Replacement

`src/lib/auth.ts` internals change. The function signature and error contract stay identical.

**New behavior:**

```
getCurrentUserId()
  → Is NEXTAUTH_SECRET set? (production mode)
    → Yes: read JWT from cookies via getToken()
      → Token exists and has sub? Return sub (userId)
      → No token / no sub? Throw AuthenticationError
    → No: fall back to seed user lookup (existing dev behavior)
```

Uses `getToken()` from `next-auth/jwt` + `cookies()` from `next/headers` to read the NextAuth cookie server-side without needing the request object.

**Contract preserved:**
- Function signature: `async function getCurrentUserId(): Promise<string>`
- Throws `AuthenticationError` on failure → controllers return 401
- Zero controller changes needed
- All existing tests pass (test env has no NEXTAUTH_SECRET → seed user fallback)

## Frontend Auth Changes

### Sign-in Page (`src/app/auth/signin/page.tsx`)

Custom styled page matching the dark theme:
- Background: `#111827`
- Centered card: `#1f2937`, `border-radius: 6px`, max-width 400px
- "Tessitura" heading (24px, `#f9fafb`, semibold)
- Subtitle: "Practice grid platform for musicians" (14px, `#6b7280`)
- "Sign in with Google" button: white background, Google logo, dark text. Standard Google branding guidelines.
- Error display: if `?error=...` query param present, show error message in `#ef4444`

Uses `signIn('google', { callbackUrl: '/' })` from `next-auth/react`.

### SessionProvider

`src/app/layout.tsx` wraps children in NextAuth's `SessionProvider` (alongside existing `QueryProvider`). Order: `SessionProvider` > `QueryProvider` > `{children}`.

Requires a new client wrapper component since `SessionProvider` is a client component: `src/lib/session-provider.tsx`.

### Route Protection (`src/middleware.ts`)

Next.js middleware checks for the NextAuth JWT cookie on protected routes:

**Protected:** `/`, `/grids`, `/grids/*`
**Excluded:** `/auth/*`, `/api/auth/*`, `/_next/*`, `/favicon.ico`

Unauthenticated requests to protected routes → redirect to `/auth/signin`.

Uses `getToken()` from `next-auth/jwt` to verify the cookie. Does NOT use the NextAuth middleware wrapper (it's simpler to write the check directly).

### Sign-out Button (`src/components/presentation/SignOutButton.tsx`)

Small presentation component: "Sign out" text link (`#6b7280`, 11px). Uses `signOut({ callbackUrl: '/auth/signin' })` from `next-auth/react`.

### Dashboard Changes

**DashboardLayout** — add a top bar or corner slot for the sign-out button.

**DashboardDirector** — read user name from `useSession()` hook. Pass `userName` prop to AlertsPane.

**AlertsPane** — accept `userName: string` prop. Replace hardcoded "Dev User" with the prop value. Falls back to "there" if no name ("Welcome back, there").

## Error Handling

| Scenario | Behavior |
|---|---|
| No Google session | Middleware redirects to `/auth/signin` |
| Expired JWT | `getCurrentUserId()` throws AuthenticationError → 401. Frontend shows "Authentication required" (existing behavior). |
| Google OAuth error | NextAuth redirects to `/auth/signin?error=OAuthAccountNotLinked` (or similar). Sign-in page displays error. |
| First-time Google user | NextAuth auto-creates User + Account via Prisma adapter. User lands on dashboard. |
| Returning Google user | NextAuth matches by provider + providerAccountId. Refreshes JWT. Dashboard shows their name. |
| Local dev (no Google creds) | `getCurrentUserId()` falls back to seed user. App works without OAuth configured. |

## Testing Strategy

**Integration tests (Vitest):**
- `getCurrentUserId()` with mocked `getToken` → returns userId from JWT
- `getCurrentUserId()` with no token → throws AuthenticationError
- `getCurrentUserId()` with no NEXTAUTH_SECRET → falls back to seed user
- Existing controller tests continue passing unchanged (seed user fallback)

**Unit tests (Vitest + RTL):**
- SignOutButton renders and is clickable
- Sign-in page renders Google button and error message
- AlertsPane renders dynamic userName

**E2E (Playwright):**
- Unauthenticated user hitting `/` → redirected to `/auth/signin`
- Sign-in page renders with Google button
- After mock auth, dashboard shows user name (mocked session)

**Coverage:** 95% minimum on all new code. Auth-options and NextAuth route are config — they may be excluded from coverage if they're pure NextAuth wiring with no custom logic beyond what's tested via integration tests.
